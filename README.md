# FreshForward - Backend

## Tech Stack
- **FastAPI** - API framework
- **PostgreSQL** - database
- **SQLAlchemy** - ORM
- **Alembic** - database migrations
- **Pydantic** - data validation/schemas
- **python-dotenv** - environment config
- **Uvicorn** - server
- **Railway** - hosting (API + Postgres)
- **Stripe** - payment processing (Checkout Sessions - platform collects payment directly; no Connect/marketplace split yet, see productscope.md)
- **bcrypt** - password hashing
- **PyJWT** - auth tokens

## Workflow - READ THIS
**Do NOT push directly to main.** Every change goes through a branch + Pull Request.

```bash
git checkout main
git pull
git checkout -b yourname/description #eg matthew/healthroute

# code here

git add .
git commit -m "clear description of change"
git push -u origin yourname/short-description
```

Then open a PR on GitHub. Get it reviewed before merging into main.

## Setup
```bash
python -m venv venv
venv\Scripts\activate # Windows
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Copy `.env.example` to `.env` and fill in real values (DB connection string, JWT secret, Stripe keys). DO NOT COMMIT `.env`.

Then apply migrations against your local Postgres:
```bash
alembic upgrade head
```

## Database migrations

Models live in `app/models.py`. After changing them, generate a migration and check the
autogenerated file before committing (autogenerate doesn't catch everything - review it):
```bash
alembic revision --autogenerate -m "describe the change"
alembic upgrade head
```

## Deploying on Railway

1. Create a new Railway project, add a **Postgres** plugin, and deploy this repo as a service
   (Railway's Nixpacks builder auto-detects Python + the `Procfile`).
2. Set these environment variables on the service (see `.env.example`):
   `DATABASE_URL` (Railway injects this automatically if you reference the Postgres plugin),
   `SECRET_KEY`, `ALGORITHM`, `ACCESS_TOKEN_EXPIRE_MINUTES`,
   `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, `FRONTEND_URL`.
3. The `Procfile`'s `web` process runs `alembic upgrade head` before starting Uvicorn, so
   migrations apply automatically on every deploy.
4. In the Stripe dashboard, add a webhook endpoint at `https://<your-railway-domain>/payments/webhook`
   subscribed to `checkout.session.completed`, `checkout.session.expired`, and
   `checkout.session.async_payment_failed`, then copy its signing secret into `STRIPE_WEBHOOK_SECRET`.
